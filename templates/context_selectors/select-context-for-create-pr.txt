**Sua Tarefa ÚNICA e ABSOLUTAMENTE RESTRITA:**
Com base na descrição da tarefa principal e no **CONTEXTO FORNECIDO** (que inclui CONTEÚDO ESSENCIAL e SUMÁRIOS de outros arquivos), selecione e **PRIORIZE** os arquivos mais relevantes para a tarefa principal.

**Tarefa Principal:** Gerar o TÍTULO e o CORPO para um Pull Request (PR) do GitHub que implementa as mudanças relacionadas à Issue `__NUMERO_DA_ISSUE__`, seguindo padrões e vinculando a issue.

**Contexto Fornecido:**
1.  **Conteúdo Essencial Pré-injetado:** Você receberá o conteúdo completo (ou sumário/truncado) de arquivos críticos como `github_issue___NUMERO_DA_ISSUE___details.json`, `git_diff_cached.txt`, `git_diff_unstaged.txt`. Analise este conteúdo diretamente.
2.  **Sumários de Outros Arquivos:** Você receberá uma lista de sumários/metadados de outros arquivos (path, type, summary, token_count), incluindo `git_log.txt`, `gh_pr_list.txt`, guias de desenvolvimento, etc. Use estas informações para decidir se o arquivo completo correspondente deve ser incluído.

**Seu Processo de Seleção Priorizada:**
1.  Entenda profundamente o objetivo da **Tarefa Principal**: gerar Título e Corpo para um PR da Issue `__NUMERO_DA_ISSUE__`.
2.  Analise o **Conteúdo Essencial Pré-injetado** para entender requisitos e mudanças.
3.  Analise os **Sumários de Outros Arquivos** para identificar contexto adicional útil.
4.  Crie uma **LISTA PRIORIZADA** de caminhos relativos (`path`) dos arquivos que você considera MAIS importantes para a IA gerar o Título e Corpo do PR. A ordem é crucial: **os mais importantes DEVEM vir primeiro.**
5.  **Arquivos a Priorizar (em ordem aproximada de importância):**
    *   Os arquivos de diff (`git_diff_*.txt` - cujo conteúdo você já recebeu).
    *   O arquivo de detalhes da Issue (`github_issue___NUMERO_DA_ISSUE___details.json` - cujo conteúdo você já recebeu).
    *   O arquivo de log Git (`git_log.txt` - use o sumário).
    *   A lista de PRs anteriores (`gh_pr_list.txt` - use o sumário para referência de estilo).
    *   Os guias de desenvolvimento (`guia_*.md`, `padroes_*.md` - use os sumários).
6.  **Objetivo de Tokens:** Tente manter o **soma** dos `token_count` dos arquivos que você selecionou (usando o valor do sumário/metadados) abaixo de aproximadamente `__BUDGET_PARA_CHAMADA_PRINCIPAL__` tokens. **PRIORIZE incluir os diffs, detalhes da issue e o log Git** (ou seus sumários) mesmo que exceda um pouco o budget.
7.  **Exclua:** Código fonte não modificado, testes, configurações gerais, dependências, e outros arquivos não diretamente necessários para *descrever o propósito e as mudanças* do PR.

**Formato de Saída OBRIGATÓRIO E ESTRITO:**
Sua resposta DEVE ser **APENAS E SOMENTE APENAS** um objeto JSON válido contendo UMA ÚNICA chave chamada `relevant_files`. O valor desta chave DEVE ser a LISTA **PRIORIZADA** (array JSON) de strings, onde cada string é o caminho relativo EXATO de um arquivo selecionado.

**Exemplo de Saída:**
```json
{
  "relevant_files": [
    "context_llm/code/YYYYMMDD_HHMMSS/git_diff_cached.txt",
    "context_llm/code/YYYYMMDD_HHMMSS/github_issue___NUMERO_DA_ISSUE___details.json",
    "context_llm/code/YYYYMMDD_HHMMSS/git_log.txt",
    "context_llm/code/YYYYMMDD_HHMMSS/gh_pr_list.txt",
    "docs/guia_de_desenvolvimento.md"
  ]
}
```

**REGRAS ADICIONAIS:**
-   **NÃO** inclua NENHUM texto explicativo.
-   Sua resposta completa DEVE começar com `{` e terminar com `}`.
-   Se NENHUM arquivo for considerado relevante (improvável), retorne `{"relevant_files": []}`.
