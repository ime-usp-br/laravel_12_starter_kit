**Sua Tarefa ÚNICA e ABSOLUTAMENTE RESTRITA:**
Com base na descrição da tarefa principal e no **CONTEXTO FORNECIDO** (que inclui CONTEÚDO ESSENCIAL e SUMÁRIOS de outros arquivos), selecione e **PRIORIZE** os arquivos mais relevantes para a tarefa principal.

**Tarefa Principal:** Corrigir os erros de análise estática reportados no arquivo `phpstan_analysis.txt`, gerando o código PHP corrigido para os arquivos afetados, seguindo os padrões do projeto.

**Contexto Fornecido:**
1.  **Conteúdo Essencial Pré-injetado:** Você receberá o conteúdo completo (ou sumário/truncado) de `phpstan_analysis.txt`. Analise este conteúdo diretamente para identificar os erros, arquivos e linhas afetadas.
2.  **Sumários de Outros Arquivos:** Você receberá uma lista de sumários/metadados de outros arquivos (path, type, summary, token_count), incluindo os arquivos PHP mencionados no log do PHPStan, guias de desenvolvimento, etc. Use estas informações para decidir se o arquivo completo correspondente deve ser incluído.

**Seu Processo de Seleção Priorizada:**
1.  Entenda profundamente o objetivo da **Tarefa Principal**: corrigir erros do PHPStan listados em `phpstan_analysis.txt`.
2.  Analise o **Conteúdo Essencial Pré-injetado** (log PHPStan).
3.  Analise os **Sumários de Outros Arquivos**.
4.  Crie uma **LISTA PRIORIZADA** de caminhos relativos (`path`) dos arquivos que você considera MAIS importantes para a IA diagnosticar e corrigir os erros. A ordem é crucial.
5.  **Arquivos a Priorizar (em ordem aproximada de importância):**
    *   O log de análise do PHPStan (`phpstan_analysis.txt` - cujo conteúdo você já recebeu).
    *   Os arquivos `.php` **exatos** mencionados no log do PHPStan como contendo erros (identifique pelos sumários).
    *   Os guias de desenvolvimento (`guia_*.md`, `padroes_*.md` - use sumários).
    *   (Opcional) Outros arquivos `.php` referenciados nos erros (para tipos/assinaturas), se os sumários indicarem relevância direta para o erro.
6.  **Objetivo de Tokens:** Tente manter o **soma** dos `token_count` dos arquivos que você selecionou (usando o valor do sumário/metadados) abaixo de aproximadamente `__BUDGET_PARA_CHAMADA_PRINCIPAL__` tokens. **PRIORIZE incluir o log PHPStan e os arquivos PHP diretamente afetados**, mesmo que exceda um pouco o budget.
7.  **Exclua:** Arquivos PHP não mencionados no log, testes (a menos que o erro esteja no teste), diffs, logs Git, issues, configurações não relevantes, dependências, Views, assets, etc.

**Formato de Saída OBRIGATÓRIO E ESTRITO:**
Sua resposta DEVE ser **APENAS E SOMENTE APENAS** um objeto JSON válido contendo UMA ÚNICA chave chamada `relevant_files`. O valor desta chave DEVE ser a LISTA **PRIORIZADA** (array JSON) de strings, onde cada string é o caminho relativo EXATO de um arquivo selecionado.

**Exemplo de Saída:**
```json
{
  "relevant_files": [
    "context_llm/code/YYYYMMDD_HHMMSS/phpstan_analysis.txt",
    "app/Http/Controllers/ControllerWithError.php",
    "app/Services/ServiceWithTypeIssue.php",
    "docs/padroes_codigo_boas_praticas.md"
  ]
}
```

**REGRAS ADICIONAIS:**
-   **NÃO** inclua NENHUM texto explicativo.
-   Sua resposta completa DEVE começar com `{` e terminar com `}`.
-   Se NENHUM arquivo for considerado relevante (improvável), retorne `{"relevant_files": []}`.
