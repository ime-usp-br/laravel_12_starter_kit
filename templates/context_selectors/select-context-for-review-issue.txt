Com base na descrição da tarefa principal e no **CONTEXTO FORNECIDO** (que inclui CONTEÚDO ESSENCIAL e SUMÁRIOS de outros arquivos), selecione e **PRIORIZE** os arquivos mais relevantes para a tarefa principal.

**Tarefa Principal:** Revisar a Issue GitHub `__NUMERO_DA_ISSUE__` e gerar um corpo Markdown atualizado e completo para ela, alinhado com o estado atual do projeto e padrões.

**Contexto Fornecido:**
1.  **Conteúdo Essencial Pré-injetado:** Você receberá o conteúdo completo (ou sumário/truncado) de `github_issue___NUMERO_DA_ISSUE___details.json`. Analise este conteúdo diretamente como base da revisão.
2.  **Sumários de Outros Arquivos:** Você receberá uma lista de sumários/metadados de outros arquivos (path, type, summary, token_count), incluindo documentação, logs, código, etc. Use estas informações para decidir se o arquivo completo correspondente deve ser incluído.

**Seu Processo de Seleção Priorizada:**
1.  Entenda profundamente o objetivo da **Tarefa Principal**: revisar e atualizar o corpo da Issue `__NUMERO_DA_ISSUE__`.
2.  Analise o **Conteúdo Essencial Pré-injetado** (detalhes da issue alvo).
3.  Analise os **Sumários de Outros Arquivos**.
4.  Crie uma **LISTA PRIORIZADA** de caminhos relativos (`path`) dos arquivos que você considera MAIS importantes para a IA revisar a issue. A ordem é crucial.
5.  **Arquivos a Priorizar (em ordem aproximada de importância):**
    *   O arquivo de detalhes da Issue alvo (`github_issue___NUMERO_DA_ISSUE___details.json` - conteúdo pré-injetado).
    *   **TODA** a documentação principal (`README.md`, todos `docs/*.md` - identifique pelos sumários).
    *   Arquivos que refletem o estado atual do projeto: `git_log.txt`, `git_status.txt`, `manifest.json` (ou `_manifest.json`), `project_tree_L*.txt`, `artisan_route_list.json`, `composer_show.txt`, `npm_list*` (identifique pelos sumários).
    *   Outras Issues e PRs recentes (`gh_pr_list.txt`, outros `github_issue_*_details.json`) que possam contextualizar (identifique pelos sumários).
    *   Código fonte (`app/...`, `resources/...`, `tests/...`) **DIRETAMENTE RELACIONADO** ao assunto da Issue `__NUMERO_DA_ISSUE__` (use os sumários para avaliar a relevância).

7.  **Exclua:** Arquivos de contexto muito antigos, diffs específicos irrelevantes, arquivos de build, configurações genéricas, código não relacionado.

**Formato de Saída OBRIGATÓRIO E ESTRITO:**
Sua resposta DEVE ser **APENAS E SOMENTE APENAS** um objeto JSON válido contendo UMA ÚNICA chave chamada `relevant_files`. O valor desta chave DEVE ser a LISTA **PRIORIZADA** (array JSON) de strings, onde cada string é o caminho relativo EXATO de um arquivo selecionado.

**Exemplo de Saída:**
```json
{
  "relevant_files": [
    "context_llm/code/YYYYMMDD_HHMMSS/github_issue___NUMERO_DA_ISSUE___details.json",

    "docs/guia_de_desenvolvimento.md",
    "docs/padroes_codigo_boas_praticas.md",
    "context_llm/code/YYYYMMDD_HHMMSS/git_log.txt",
    "context_llm/code/YYYYMMDD_HHMMSS/manifest.json",
    "app/Http/Controllers/MaybeRelatedController.php"
  ]
}
```

**REGRAS ADICIONAIS:**
-   **NÃO** inclua NENHUM texto explicativo.
-   Sua resposta completa DEVE começar com `{` e terminar com `}`.
-   Se NENHUM arquivo for considerado relevante (improvável), retorne `{"relevant_files": []}`.
