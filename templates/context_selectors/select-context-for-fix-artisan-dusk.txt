**Sua Tarefa ÚNICA e ABSOLUTAMENTE RESTRITA:**
Com base na descrição da tarefa principal e no **CONTEXTO FORNECIDO** (que inclui CONTEÚDO ESSENCIAL e SUMÁRIOS de outros arquivos), selecione e **PRIORIZE** os arquivos mais relevantes para a tarefa principal.

**Tarefa Principal:** Analisar a saída do comando `php artisan dusk` (contida em `dusk_test_results.txt`), identificar os testes que falharam, determinar a causa e gerar o código corrigido para o(s) arquivo(s) apropriado(s) (Teste Dusk, Blade/Livewire, JS).

**Contexto Fornecido:**
1.  **Conteúdo Essencial Pré-injetado:** Você receberá o conteúdo completo (ou sumário/truncado) de `dusk_test_results.txt`. Analise este conteúdo diretamente para identificar as falhas.
2.  **Sumários de Outros Arquivos:** Você receberá uma lista de sumários/metadados de outros arquivos (path, type, summary, token_count), incluindo arquivos de teste Dusk, views Blade/Livewire, JS, guias, etc. Use estas informações para decidir se o arquivo completo correspondente deve ser incluído.

**Seu Processo de Seleção Priorizada:**
1.  Entenda profundamente o objetivo da **Tarefa Principal**: corrigir falhas reportadas em `dusk_test_results.txt`.
2.  Analise o **Conteúdo Essencial Pré-injetado** (log Dusk).
3.  Analise os **Sumários de Outros Arquivos**.
4.  Crie uma **LISTA PRIORIZADA** de caminhos relativos (`path`) dos arquivos que você considera MAIS importantes para a IA diagnosticar e corrigir as falhas. A ordem é crucial.
5.  **Arquivos a Priorizar (em ordem aproximada de importância):**
    *   O log de resultados Dusk (`dusk_test_results.txt` - cujo conteúdo você já recebeu).
    *   Os arquivos de teste Dusk (`tests/Browser/...Test.php`) específicos mencionados como falhos no log (identifique pelos sumários).
    *   Os arquivos de View Blade/Livewire (`resources/views/...blade.php`) que os testes falhos estavam interagindo (identifique pelos sumários/nomes de teste).
    *   Arquivos JavaScript (`resources/js/...`) se os erros indicarem problemas de JS (identifique pelos sumários).
    *   Guias de desenvolvimento (`guia_*.md`, `padroes_*.md` - use sumários).
6.  **Objetivo de Tokens:** Tente manter o **soma** dos `token_count` dos arquivos que você selecionou (usando o valor do sumário/metadados) abaixo de aproximadamente `__BUDGET_PARA_CHAMADA_PRINCIPAL__` tokens. **PRIORIZE incluir o log Dusk, o teste falho e a view/componente associado**, mesmo que exceda um pouco o budget.
7.  **Exclua:** Testes PHPUnit, código backend não relacionado à UI, logs Git, issues, configurações não relevantes, dependências, etc.

**Formato de Saída OBRIGATÓRIO E ESTRITO:**
Sua resposta DEVE ser **APENAS E SOMENTE APENAS** um objeto JSON válido contendo UMA ÚNICA chave chamada `relevant_files`. O valor desta chave DEVE ser a LISTA **PRIORIZADA** (array JSON) de strings, onde cada string é o caminho relativo EXATO de um arquivo selecionado.

**Exemplo de Saída:**
```json
{
  "relevant_files": [
    "context_llm/code/YYYYMMDD_HHMMSS/dusk_test_results.txt",
    "tests/Browser/FailingDuskTest.php",
    "resources/views/livewire/problematic/component.blade.php",
    "resources/js/relevant_script.js",
    "docs/padroes_codigo_boas_praticas.md"
  ]
}
```

**REGRAS ADICIONAIS:**
-   **NÃO** inclua NENHUM texto explicativo.
-   Sua resposta completa DEVE começar com `{` e terminar com `}`.
-   Se NENHUM arquivo for considerado relevante (improvável), retorne `{"relevant_files": []}`.
