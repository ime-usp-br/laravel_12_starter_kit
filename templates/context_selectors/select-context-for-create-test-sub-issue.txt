**Sua Tarefa ÚNICA e ABSOLUTAMENTE RESTRITA:**
Com base na descrição da tarefa principal e no **CONTEXTO FORNECIDO** (que inclui CONTEÚDO ESSENCIAL e SUMÁRIOS de outros arquivos), selecione e **PRIORIZE** os arquivos mais relevantes para a tarefa principal.

**Tarefa Principal:** Gerar o texto completo (formato `KEY: VALUE`) para um arquivo de plano `.txt`, definindo uma sub-issue do GitHub focada em testes (Unit, Feature, Dusk) e CI para a funcionalidade da Issue pai `#__PARENT_ISSUE_NUMBER__`.

1.  **Conteúdo Essencial Pré-injetado:** Você receberá o conteúdo completo (ou sumário/truncado) do arquivo de detalhes da Issue pai: `github_issue___PARENT_ISSUE_NUMBER___details.json`. Analise este conteúdo diretamente para entender a funcionalidade a ser testada.
2.  **Sumários de Outros Arquivos:** Você receberá uma lista de sumários/metadados de outros arquivos (path, type, summary, token_count), incluindo código fonte relevante, testes existentes, workflow CI, guias de desenvolvimento, etc. Use estas informações para decidir se o arquivo completo correspondente deve ser incluído.

**Seu Processo de Seleção Priorizada:**
1.  Entenda profundamente o objetivo da **Tarefa Principal**: gerar a definição de uma sub-issue de teste para a Issue `#__PARENT_ISSUE_NUMBER__`.
2.  Analise o **Conteúdo Essencial Pré-injetado** (detalhes da issue pai).
3.  Analise os **Sumários de Outros Arquivos** para identificar contexto adicional.
4.  Crie uma **LISTA PRIORIZADA** de caminhos relativos (`path`) dos arquivos que você considera MAIS importantes para a IA gerar a definição da sub-issue de teste. A ordem é crucial.
5.  **Arquivos a Priorizar (em ordem aproximada de importância):**
    *   O arquivo de detalhes da Issue pai (`github_issue___PARENT_ISSUE_NUMBER___details.json` - cujo conteúdo você já recebeu).
    *   Arquivos de código fonte (`app/...`, `resources/views/...`) que implementam a funcionalidade da issue pai (identifique pelos sumários).

    *   Os guias de desenvolvimento/padrões de teste (`guia_*.md`, `padroes_*.md` - use sumários).
